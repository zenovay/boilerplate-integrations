## Install

Apply the patch from the upstream repository root:

```bash
git apply 0001-feat-analytics-add-optional-Zenovay-provider.patch
```

The patch is in [`patches/`](patches/). It adds
`@zenovay/tracker@0.1.0` to `packages/analytics/package.json`. It extends
`packages/analytics/keys.ts` with an optional client key named
`NEXT_PUBLIC_ZENOVAY_TRACKING_CODE`. In
`packages/analytics/instrumentation-client.ts`, initialize Zenovay alongside
PostHog:

```ts
import { init } from '@zenovay/tracker'

const code = process.env.NEXT_PUBLIC_ZENOVAY_TRACKING_CODE
if (code) init(code, { cookieless: true })
```

Add the public variable to each app that imports `@repo/analytics` and to the
deployment environment. Keep PostHog, Google, and Vercel providers available;
this is an additional provider, not a forced replacement.

## Page views and SPA navigation

The initial page is automatic. Add a small client component inside the existing
`AnalyticsProvider` that uses `usePathname`/`useSearchParams`, stores the
initial URL, and calls `page()` only after a unique client navigation.

## Events

Bridge next-forge application actions to the public helpers:

```ts
track('checkout_started', { plan: planId })
goal('signup_completed')
identify(user.id, { email: user.email })
revenue(49, 'USD', { order_id: checkoutSession.id })
```

Connect Stripe in Zenovay for authoritative server-side revenue attribution;
never expose Stripe secrets in browser code.

## Privacy and first-party tracking

The adapter is cookieless. If replay requires consent, call
`window.zenovay?.('consent', 'granted')` from the existing consent system. First-party tracking uses the custom-domain values
generated by Zenovay, not next-forge's PostHog `/ingest` rewrite.

## Local development and production

Enable Allow Localhost for the Zenovay site. Add the public code to all three
Next.js app environments that load analytics. Update CSP if customized.

## Troubleshooting

- Initialization absent: the app must import its existing instrumentation client.
- Double views: keep a single provider and skip the initial router callback.
- No local events: enable the site setting.
- First-party requests still use the default host: copy the generated dashboard configuration.

## Removal

Remove the dependency, key schema entry, initialization and navigation
component, environment variables, event calls, and Zenovay CSP entries.
