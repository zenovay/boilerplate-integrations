## Install

Add a `Zenovay.tsx` provider beside the existing analytics components:

```tsx
import Script from 'next/script.js'

export interface ZenovayProps {
  trackingCode: string
  cookieless?: boolean
  src?: string
}

export const Zenovay = ({
  trackingCode,
  cookieless = true,
  src = 'https://api.zenovay.com/z.js',
}: ZenovayProps) => (
  <Script
    id="zenovay-script"
    strategy="afterInteractive"
    data-id={trackingCode}
    data-cookieless={cookieless ? 'true' : undefined}
    src={src}
  />
)
```

Export it from `analytics/index.tsx`, add `zenovayAnalytics?: ZenovayProps` to
`AnalyticsConfig`, and render it in production. Consumers set the public code
in `siteMetadata.analytics.zenovayAnalytics.trackingCode`.

## Page views and SPA navigation

The script handles the initial page. A provider enhancement should use Next.js
navigation hooks and a previous-URL ref for later unique navigations. It must
not emit from server rendering.

## Events

Pliny consumers call the global API from client code:

```ts
window.zenovay?.('track', 'newsletter_subscribed')
window.zenovay?.('goal', 'signup_completed')
window.zenovay?.('identify', user.id, { email: user.email })
window.zenovay?.('revenue', 49, 'USD', { order_id: order.id })
```

## Privacy and first-party tracking

Cookieless is the default in this adapter. A consumer can pass the verified
custom-domain script URL generated by Zenovay on eligible plans.

## Local development and production

Pliny renders analytics only in production. Set the public code for production
and use a production-mode preview for verification. Enable Allow Localhost if
testing events locally. Add Zenovay to the consumer's CSP.

## Verification

Run Pliny's package build and typecheck, then consume the built package from the
Tailwind Starter Blog. Confirm the generated page contains one script.

## Troubleshooting

- Provider not rendered in development: Pliny's analytics switch is production-only.
- CSP blocked: add the tracker and event origins.
- Duplicate script: enable one provider instance only.
- No local event: enable Allow Localhost.

## Removal

Delete the provider/export/config field, remove consumer metadata and CSP
entries, and remove event calls.
