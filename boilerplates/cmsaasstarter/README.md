## Install

Apply the patch from the upstream repository root:

```bash
git apply 0001-feat-add-optional-Zenovay-analytics.patch
```

The patch is in [`patches/`](patches/). Set
`PUBLIC_ZENOVAY_TRACKING_CODE=ZV_YOUR_CODE` to enable it. It initializes the
public tracker in the root layout and records SvelteKit client navigation with
`afterNavigate`. Nothing is loaded when the value is absent.

## Events

Authenticated account pages identify the stable Supabase user so acquisition
can be connected to activation. Connect the project's Stripe account in
Zenovay for authoritative subscription revenue instead of inferring payment
success from the browser redirect that creates Checkout.

## Privacy and first-party tracking

The integration explicitly enables cookieless mode. Use the first-party script
configuration generated by the Zenovay dashboard when available. Do not add
profile fields or Stripe details as analytics properties.

## Local development and production

Local event ingestion is controlled by the Zenovay website setting. The
starter's static Supabase and Stripe imports require a local `.env`, which can
be copied from `.env.example` for compilation. Use real secrets only in local
ignored files or the deployment environment.

## Troubleshooting

- Missing environment exports: copy `.env.example` to `.env` before checking.
- No pageviews: confirm the public tracking code is available to the client.
- No local events: enable local ingestion for the isolated Zenovay site.
- No identity: verify the authenticated account layout receives `data.user`.

## Removal

Reverse the patch or remove the tracker dependency, root layout hooks, account
identity call, public environment value, and analytics documentation section.
