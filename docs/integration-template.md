# Integration template

Copy this file when adding a framework or boilerplate integration. Remove
sections only when they are genuinely not applicable.

## Status

- Status: `verified | community | guide-only | planned | blocked-private | deprecated`
- Upstream version or commit:
- Last verified: `YYYY-MM-DD`
- Maintainer:
- Upstream documentation:
- Verification evidence:

## Install

List exact package commands, file paths, environment variables, and code. A
tracking code is public browser configuration, not a secret, but always use a
placeholder in this repository.

## Page views and SPA navigation

Explain whether the tracker captures the initial page view automatically and
how subsequent client-side navigations are handled without duplicates.

## Events

Include:

- a custom event with `track`;
- a signup conversion with `goal`;
- a stable authenticated-user ID with `identify`;
- revenue or checkout with `revenue` when relevant.

## Privacy and first-party tracking

Show explicit cookieless configuration. For first-party tracking, tell users
to copy the custom-domain snippet generated by the Zenovay dashboard; do not
guess proxy routes.

## Local development and production

Explain the dashboard Allow Localhost setting, build/deploy variables, CSP, and
verification in browser DevTools and Zenovay Live View.

## Troubleshooting

Cover script blocking, wrong tracking code, CSP, duplicate install, SPA double
counting, localhost settings, and delayed events.

## Removal

List every file, package, environment variable, CSP entry, and dashboard
setting to remove.
